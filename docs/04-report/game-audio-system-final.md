# PDCA Report: 게임 오디오 시스템 (Game Audio System)

## 1. 프로젝트 개요 (Project Overview)
스택 타워 게임의 사용자 몰입도를 높이기 위해 Web Audio API를 활용한 실시간 오디오 합성 시스템을 구축하였습니다. 별도의 오디오 자산 파일 없이 순수 코드로 효과음을 생성하여 성능과 로딩 속도를 동시에 확보하였습니다.

## 2. 수행 결과 (Execution Results)

### 2.1. 주요 성과 (Key Achievements)
- **자립형 오디오 매니저**: `AudioManager` 클래스를 구현하여 브라우저 내장 오실레이터를 제어, 외부 mp3/wav 파일 의존성 제거.
- **다이내믹 피치 연출**: 블록이 쌓일수록 주파수가 올라가는 로직을 적용하여 게임 후반부의 긴장감을 시각적 요소와 동기화.
- **상황별 음색 설계**: 
  - 일반 배치: Triangle 파형 (부드러운 타격감)
  - 퍼펙트 매치: Sine 파형 (맑고 높은 피드백)
  - 게임 오버: Sawtooth 파형 (하강하는 Slide 효과)
- **UX 최적화**: 브라우저의 오디오 자동 재생 차단 정책(Autoplay Policy)을 준수하여 사용자 첫 클릭 시점에 시스템 활성화.

### 2.2. 기술적 과제 해결
- **틱 노이즈 방지**: `exponentialRampToValueAtTime`을 사용하여 소리의 끝부분을 부드럽게 감쇠시켜 부자연스러운 클릭 노이즈 제거.
- **리소스 최적화**: 오디오 노드들을 필요 시점에 동적으로 연결하고 해제하여 시스템 자원 소모 최소화.

## 3. 최종 분석 (Final Analysis)
- **달성률**: 100%
- **성능 영향**: 오디오 처리에 의한 프레임 드랍(FPS Drop) 현상 없음.
- **몰입도 향상**: 시각적 피드백에 청각적 피드백이 더해져 게임의 타격감이 대폭 개선됨.

## 4. 향후 계획 (Future Roadmap)
- **배경음악 합성**: 오실레이터를 활용한 간단한 8비트 스타일의 배경음악(BGM) 루프 추가.
- **오디오 설정**: 플랫폼 UI에 사운드 On/Off 또는 볼륨 조절 기능 통합.

## 5. 결론 (Conclusion)
게임 오디오 시스템 도입을 통해 '하이퍼 게임'이라는 명칭에 걸맞은 완성도 높은 사용자 경험을 제공하게 되었습니다. 본 시스템은 향후 플랫폼 내 다른 게임들에도 쉽게 이식 가능한 구조로 설계되었습니다.
