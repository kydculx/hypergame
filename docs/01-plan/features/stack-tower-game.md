# PDCA Plan: 스택 타워 게임 (Stack Tower Game)

## 1. 개요 (Overview)
Silvergames 등에서 인기 있는 "Stack" 류의 하이퍼 캐주얼 게임을 개발합니다. 좌우로 움직이는 블록을 타이밍에 맞춰 쌓아 올리는 원버튼(One-button) 게임입니다.

## 2. 목표 (Goals)
- **중독성 있는 원버튼 플레이**: 클릭/탭 한번으로 즐길 수 있는 단순하지만 깊이 있는 조작감 구현.
- **시각적 만족감**: 블록이 정확히 맞았을 때의 효과와 쌓여가는 타워의 시각적 연출.
- **플랫폼 연동**: 하이퍼 게임 플랫폼의 점수 시스템과 완벽하게 연동.

## 3. 핵심 기능 (Key Features)
- **블록 이동**: 현재 쌓아야 할 블록이 좌우(또는 전후)로 왕복 이동.
- **스택 메카니즘**:
  - 클릭 시 블록 정지 및 배치.
  - 이전 블록과 겹치지 않은 부분은 잘려나가며 떨어지는 물리 효과(또는 시각 효과).
  - 겹치는 영역이 없으면 게임 오버.
- **난이도 상승**: 타워가 높아질수록 블록 이동 속도 증가.
- **점수 시스템**: 층수가 곧 점수가 되며, 플랫폼으로 실시간 전송 (`SUBMIT_SCORE`).

## 4. 기술 스택 (Technical Stack)
- **Engine**: Three.js (3D 그래픽) 또는 Vanilla JS + Canvas (2D). *입체감을 위해 Three.js 권장.*
- **Physics**: 간단한 AABB(Axis-Aligned Bounding Box) 충돌 처리.
- **Audio**: 간단한 효과음 (블록 배치, 게임 오버).

## 5. 로드맵 (Roadmap)
1. **Phase 1: 기본 씬 구성** - Three.js 기본 세팅 (카메라, 조명, 바닥).
2. **Phase 2: 블록 이동 및 배치** - 왕복 운동 로직 및 클릭 시 정지 로직 구현.
3. **Phase 3: 절단 알고리즘** - 겹친 부분 계산 및 나머지 부분 절단 처리.
4. **Phase 4: 게임 루프 및 연동** - 게임 오버 처리, 점수 계산, 플랫폼 통신(`postMessage`).
5. **Phase 5: 폴리싱** - 색상 변화, 카메라 상승 효과, UI 다듬기.

## 6. 성공 지표 (Success Metrics)
- 모바일 및 데스크탑에서 60fps 유지.
- 플랫폼의 iframe 내에서 정상 작동 및 점수 전송 확인.
