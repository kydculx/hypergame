# PDCA Plan: 사용자 프로필 시스템 (User Profile System)

## 1. 개요 (Overview)
기존의 고정된 'Guest' 이름 대신 사용자가 직접 자신의 닉네임을 설정하고 관리할 수 있는 프로필 시스템을 구축합니다. 이를 통해 랭킹 시스템에 자신의 이름을 기록하고 개인화된 경험을 제공합니다.

## 2. 목표 (Goals)
- **닉네임 설정**: 사용자가 자유롭게 이름을 변경할 수 있는 UI 제공.
- **데이터 지속성**: 설정한 닉네임이 브라우저 재방문 시에도 유지되도록 저장.
- **랭킹 연동**: 점수 기록 시 'Guest' 대신 사용자가 설정한 닉네임으로 저장.
- **UI 개인화**: 플랫폼 헤더나 로비에 환영 메시지와 함께 현재 닉네임 표시.

## 3. 핵심 기능 (Key Features)
- **프로필 편집 모달/섹션**: 로비 페이지 상단에 닉네임을 수정할 수 있는 입력 창 구현.
- **Zustand 상태 확장**: `userProfile` 상태를 추가하고 닉네임 변경 액션 구현.
- **로컬 스토리지 연동**: `persist` 미들웨어를 확장하여 프로필 정보 저장.
- **랭킹 반영 로직 수정**: 점수 추가 시 `get().userProfile.name`을 사용하여 기록 생성.

## 4. 기술 스택 (Technical Stack)
- **State Management**: Zustand (Persistence Middleware).
- **UI**: Tailwind CSS, Lucide React (User, Edit 아이콘).

## 5. 로드맵 (Roadmap)
1. **Phase 1: 상태 관리 확장** - `useGameStore`에 `userProfile` 객체 및 `setNickname` 액션 추가.
2. **Phase 2: 프로필 UI 구현** - 로비 상단에 현재 닉네임 표시 및 수정을 위한 팝오버/모달 추가.
3. **Phase 3: 랭킹 시스템 연동** - `addScore` 액션 호출 시 현재 설정된 닉네임을 사용하도록 수정.
4. **Phase 4: 유효성 검사** - 너무 길거나 짧은 이름, 부적절한 언어 필터링 등 기초적인 검증 추가.
5. **Phase 5: 폴리싱** - 닉네임 변경 시 성공 알림(Toast) 연출.

## 6. 성공 지표 (Success Metrics)
- 사용자가 닉네임을 변경하면 즉시 플랫폼 상단에 반영됨.
- 새로운 게임 점수 기록 시 변경된 닉네임으로 리더보드에 표시됨.
- 페이지 새로고침 후에도 변경된 닉네임이 유지됨.
