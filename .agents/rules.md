# 에이전트 규칙 (Agent Rules)

## 언어 및 작업 규칙
- 모든 대화와 기술적 설명은 **한국어**로 진행합니다.
- **표준 작업 프로세스**: 모든 작업은 반드시 **[계획] -> [개발] -> [검증]** 단계를 거칩니다.
  1. **계획 (Planning)**: 요구사항을 분석하고 `implementation_plan.md`를 작성하여 승인을 받습니다.
  2. **개발 (Execution)**: 확정된 계획에 따라 고품질의 코드와 UI를 구현합니다.
  3. **검증 (Verification)**: 구현된 기능이 정상 작동하는지 테스트하고 `walkthrough.md`로 결과를 보고합니다.
- **게임 개발 시 주의사항**: 새로운 게임을 만들거나 수정할 때는 반드시 `.agents/workflows/add_new_game.md` 가이드를 먼저 읽고 그 규격에 맞춰 개발합니다.
- **코드 주석**: 모든 코드 주석은 반드시 **한글**로 작성합니다.

## 디자인 원칙
- 사용자 요청에 따른 웹 개발 시, 네온(Neon) 및 글래스모피즘(Glassmorphism) 스타일을 우선적으로 고려합니다.
- 현대적이고 프리미엄한 UI/UX를 지향합니다.
