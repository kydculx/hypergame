# Web Game Balancing Skill

이 스킬은 게임이 "너무 쉽거나", "너무 어렵거나", "재미가 없을 때" 에이전트(AI)가 마치 전문 레벨 디자이너이자 밸런싱 엔지니어처럼 행동하도록 안내합니다.

## 🛠 주요 밸런싱 절차 (에이전트 행동 지침)

### 1. 전역 설정(CONFIG) 분석
- `grep_search` 도구를 사용하여 `CONFIG` 객체나 `speed`, `spawnRate`, `gap` 관련 변수를 모두 찾습니다.
- 현재 게임의 난이도 증가 방식이 선형(Linear)인지 지수(Exponential) 방식인지 수학적으로 분석합니다. (예: `Math.pow(progress, 2)`)

### 2. 난이도 곡선(Difficulty Curve) 시뮬레이션
- 플레이어의 진행도(`score` 또는 `distance`)에 따른 속도 변화와 장애물 빈도표를 작성하여 사용자에게 가시적으로 제시합니다.
- 예: "1,000점일 때는 장애물 간격이 10m, 5,000점일 때는 3m입니다."처럼 체감 난이도를 계산하여 보여줍니다.

### 3. 아이템 및 보상 티어 조정
- 아이템 스폰 확률과 페널티 비율이 합리적인지 검토합니다.
- 무적(Shield), 부스트(Boost), 동전(Coin) 같은 아이템이 과도하게 중첩되어 게임의 긴장감을 해치지 않는지 파악하고 확률을 너프(Nerf)하거나 버프(Buff)합니다.

## ✨ 밸런싱 완료 기준
- 게임 초반(0~30초)은 누구나 쉽게 접근할 수 있도록 튜토리얼 수준인가?
- 중반(30초~1분)부터 점진적으로 텐션이 올라가는가?
- 후반(1분 이상)에서는 플레이어의 극한의 컨트롤을 요구할 만큼 충분히 아드레날린이 분비되는 레벨 디자인이 적용되었는가?
