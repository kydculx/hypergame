# Web Game Asset Optimization Skill

이 스킬은 웹에서 구동되는 미니 게임이 모바일 네트워크(3G/LTE) 환경에서도 1초 이내에 로딩될 수 있도록, 에이전트(AI)가 리소스 최적화 전문가처럼 행동하도록 안내합니다.

## 🛠 주요 감사 절차 (에이전트 행동 지침)

### 1. 미디어 파일 스캔 및 용량 분석
- `find_by_name` 도구를 사용하여 프로젝트 내의 `.png`, `.jpg`, `.mp3`, `.wav`, `.gltf`, `.bin` 파일들을 모두 검색합니다.
- 파일 크기가 웹 게임 기준에 과도한지(예: 이미지 500KB 이상, 오디오 1MB 이상) 식별합니다.

### 2. 포맷 변환 및 압축 제안
- 비트맵 이미지(PNG/JPG) 중 단순한 UI나 아이콘은 **SVG 코드**로 대체하여 HTML 내부에 인라인(Inline)시키도록 코드를 리팩토링합니다.
- 용량이 큰 오디오 코드는 Web Audio API의 기본 Oscillator(신디사이저) 조합으로 동적 생성(Procedural Generation)할 수 있는지 검토하고, 공용 오디오 관리자 클래스로 마이그레이션 합니다.

### 3. 리소스 프리로딩(Preloading) 점검
- 게임 진입 전 레이턴시를 줄이기 위해 HTML `<head>`에 `<link rel="preload">`가 잘 적용되어 있는지 확인합니다.
- 텍스처 로딩 로직이 비동기(`async/await`)로 처리되어 있어 로딩 중 화면 병목을 일으키지 않는지 코드(`TextureLoader`)를 점검합니다.

## ✨ 최적화 완료 기준
- 파일 크기를 줄일 수 있는 경우 무손실/손실 압축 포맷(WebP, OGG) 사용을 제안했는가?
- 오디오나 텍스처 로딩이 플레이어의 초기 `START` 버튼 클릭 흐름을 방해하지 않도록 구조화되어 있는가?
